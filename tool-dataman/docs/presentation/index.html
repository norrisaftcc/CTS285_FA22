Project Overview

Dataman is a comprehensive math problem solver and trainer application built as part of the CTS285 course for Fall 2022. The project demonstrates modern software architecture with a modular design and multiple interface options, allowing users to practice and improve their math skills.

Inspired by classic educational toys like "Little Professor" and "Dataman," this application brings math practice into the digital age with advanced features like personalized learning, performance tracking, and adaptive difficulty levels.

Application Design

Core Architecture

Dataman follows a modular design with clear separation of concerns:

  • Core Models: Problem and ProblemSet classes that encapsulate math operations
  • Storage Interface: Flexible persistence with JSON and SQLite implementations
  • Operations: Business logic for problem management, validation, and statistics
  • User History: Comprehensive tracking of user performance and achievements
  • UI Interfaces: Multiple interfaces including CLI and Streamlit web app

Technical Highlights

Factory Pattern

Uses the factory pattern for creating appropriate storage implementations, allowing for flexible and extensible data persistence.

Decorator Pattern

Implements decorators for history tracking, allowing clean integration of user history with core functionality.

Singleton Pattern

Utilizes the singleton pattern for user history to ensure consistent access to history data throughout the application.

Demo & Examples

Answer Checker

# Example of checking a problem from dataman.core.models import Problem # Create a new problem problem = Problem(2, "+", 2) # Check if the answer is correct is_correct = problem.check_answer(4) print(f"Is 4 the correct answer? {is_correct}") # True # Try an incorrect answer is_incorrect = problem.check_answer(5) print(f"Is 5 the correct answer? {is_incorrect}") # False # The actual answer print(f"The correct answer is {problem.solve()}") # 4

Using Problem Sets

# Example of creating and using a problem set from dataman.core.models import Problem, ProblemSet from dataman.core.operations import DatamanOperations from dataman.core.storage_factory import StorageFactory # Create storage with the factory storage = StorageFactory.create_storage("json", file_path="problems.json") operations = DatamanOperations(storage) # Generate a random problem set problem_set = operations.generate_problem_set( name="Addition Practice", num_problems=5, difficulty="easy", operators=["+"] ) # Save the problem set operations.save_current_problem_set() print(f"Created problem set '{problem_set.name}' with {len(problem_set.problems)} problems")
